# AQUA Agent Guidelines

## Core Directives

**Code reuse, modularity, minimal dependencies, lazy evaluation, and performance are non-negotiable.**

Always explore `aqua/core/` before writing new code. Prefer composing existing classes over reimplementing.

---

## Architecture Rules

- **Modularity over **: when not possible to delegate to specialized (`FixerConfigure`, `FixerOperator`, `FixerDataModel`) sub-classes,  use mixins (`InstallMixin`, `CatalogMixin`, `FilesMixin`);
- **No `print()`**: always use `self.logger = log_configure(log_level=loglevel, log_name="ClassName")` and pass `loglevel` through every constructor
- **No hardcoded paths or defaults**: all paths go through `ConfigPath`; configuration is loaded once and reused
- **No new external dependencies** without discussion; check numpy/xarray/pandas/AQUA utils first
- **Respect code structure**: extend different features maintaining modularity checking respective folders (e.g. `regrid`, `reader`, `fixer`)

---

## Reuse Map

Before writing any utility, check these first:

| Need | Import |
|------|--------|
| Logging | `from aqua.core.logger import log_configure, log_history` |

## Data Handling

- Set `xr.set_options(keep_attrs=True)` once at module level; never strip attributes manually
- All xarray operations are lazy by default — **never force `.compute()` early**
- Pass `chunks` to `Reader` for dask
- Cache expensive results (areas, weights) with `hasattr` guard:
  ```python
  if not hasattr(self, '_cached_areas'):
      self._cached_areas = compute_areas(...)
  return self._cached_areas
  ```

## New Module Template

```python
from aqua.core.logger import log_configure
from aqua.core.util import to_list

class MyProcessor:
    def __init__(self, loglevel="WARNING"):
        self.logger = log_configure(log_level=loglevel, log_name="MyProcessor")
        self.loglevel = loglevel

    def process(self, data, variables=None):
        variables = to_list(variables)
        self.logger.info("Processing %d variables", len(variables))
        return data  # keep lazy
```

- Extend `Reader` to the corresponding class when the logic belongs to the read/fix/regrid chain
- CLI commands go in `aqua/core/console/` and register in `AquaConsole.command_map`
- Tests use existing fixtures in `aqua-data/` (`cell_area_*.nc`, `weights_*.nc`); never create new test data files

## Code Style

All code is linted with **Ruff**. Generated code must be Ruff-compliant: no unused imports, no bare `except`, consistent quoting, line length ≤ 127. Run `ruff check --fix` before submitting.

---

## External Dependencies

Prefer in this order before adding anything new: `numpy`, `xarray`, `pandas`, `metpy`, `smmregrid`, `intake_xarray`.

---

## Docstrings & Documentation

Use Google-style docstrings. Minimum required:

```python
def method(self, arg1, arg2=None):
    """One-line description.

    Args:
        arg1 (type): Description.
        arg2 (type, optional): Description. Defaults to None.

    Returns:
        type: Description.

    Raises:
        NoDataError: When no data is found.
    """
```

- Private/internal methods (`_name`) need only a one-liner
- Always add a usage example in the module-level docstring for public classes

---

## Testing

- Tests live in `tests/`; fixtures in `aqua-data/` (`cell_area_*.nc`, `weights_*.nc`)
- Never create new test data files — parametrize over existing grids (`test-r2b0`, `test-pi-2d`, `test-healpix`, `test-hpz3-nested`)
- Use `Reader(model=..., exp=..., source=...)` to load test data; avoid raw `xr.open_dataset` unless testing I/O directly

---

## Integration Checklist

- [ ] `log_configure()` used, `loglevel` accepted in constructor
- [ ] All paths via `ConfigPath`; no hardcoded strings
- [ ] `to_list()` used for any variable-length input
- [ ] `DataModel.apply()` used for coordinate transformations
- [ ] `xr.set_options(keep_attrs=True)` set; no `copy(deep=True)` unless necessary
- [ ] Operations remain lazy until explicitly needed
- [ ] No new dependency added
- [ ] Tests reuse `aqua-data/` fixtures
