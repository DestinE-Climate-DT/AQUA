# AQUA Agent Guidelines

## Core Directives

**Code reuse, modularity, minimal dependencies, lazy evaluation, and performance are non-negotiable.**

Always explore `aqua/core/` before writing new code. Prefer composing existing classes over reimplementing.

---

## Architecture Rules

- **Modularity over monoliths**: when not possible to delegate to specialized (`FixerConfigure`, `FixerOperator`, `FixerDataModel`) sub-classes, use mixins (`InstallMixin`, `CatalogMixin`, `FilesMixin`)
- **No `print()`**: always use `self.logger = log_configure(log_level=loglevel, log_name="ClassName")` and pass `loglevel` through every constructor
- **No hardcoded paths or ad hoc config discovery**: route installation/catalog/machine resolution through `ConfigPath` and `ConfigLocator`; respect `AQUA_CONFIG` and `$HOME/.aqua` instead of reimplementing search logic
- **Configuration is declarative**: define new grids, fixes, data models, catalogs, and analyses in YAML (`load_yaml`, `load_multi_yaml`) first; add Python `if/else` only when config cannot express the behavior
- **Protect shared YAML writes**: config/catalog/grid mutations go through `SafeFileLock` and `dump_yaml()`; avoid raw `open(..., "w")` on shared metadata files
- **No new external dependencies** without discussion; check numpy/xarray/pandas/AQUA utils first
- **Respect code structure**: extend different features maintaining modularity checking respective folders (e.g. `regrid`, `reader`, `fixer`)
- **Reader as orchestrator**: read/fix/datamodel/regrid/stats logic should be composed through `Reader` collaborators, not duplicated in standalone flows
- **Accessor contract**: if a method returns an xarray object intended for chained AQUA operations, call `data.aqua.set_default(self)` before returning
- **Prefer imports at the top of the file**: avoid importing modules within functions or methods unless strictly necessary

## Data Handling

- Set `xr.set_options(keep_attrs=True)` once at module level; never strip attributes manually
- All xarray operations are lazy by default â€” **never force `.compute()` early**
- Pass `chunks` to `Reader` for dask
- In read paths, prefer lazy loaders (`to_dask`, chunked reads) and keep chunk/aggregation handling centralized in `Reader` unless it is strictly necessary to do otherwise
- Preserve provenance and AQUA metadata: when a transform changes variables, coordinates, grids, or statistics, append `log_history(...)` and keep `AQUA_*` attrs updated via `set_attrs()` / `update_metadata()` instead of rebuilding attrs manually unless it is strictly necessary to do otherwise
- Cache expensive results (areas, weights) with `hasattr` guard:
  ```python
  if not hasattr(self, '_cached_areas'):
      self._cached_areas = compute_areas(...)
  return self._cached_areas
  ```

## New Module Template

```python
from aqua.core.logger import log_configure
from aqua.core.util import to_list

class MyProcessor:
    def __init__(self, loglevel="WARNING"):
        self.logger = log_configure(log_level=loglevel, log_name="MyProcessor")
        self.loglevel = loglevel

    def process(self, data, variables=None):
        variables = to_list(variables)
        self.logger.info("Processing %d variables", len(variables))
        return data  # keep lazy
```

- Extend `Reader` to the corresponding class when the logic belongs to the read/fix/regrid chain
- CLI commands go in `aqua/core/console/` and register in `AquaConsole.command_map`
- CLI parser definitions stay in `aqua/core/console/parser.py` (unless specific needs arise); execution logic stays in dedicated command modules or mixins
- Tests use existing fixtures in `AQUA_tests/` together with the top-level `cell_area_*.nc` and `weights_*.nc`; never create new test data files

## Docstrings & Documentation

Use Google-style docstrings. Minimum required:

```python
def method(self, arg1, arg2=None):
    """One-line description.

    Args:
        arg1 (type): Description.
        arg2 (type, optional): Description. Defaults to None.

    Returns:
        type: Description.

    Raises:
        NoDataError: When no data is found.
    """
```

- Private/internal methods (`_name`) need only a one-liner
- Always add a usage example in the module-level docstring for public classes

## Testing

- Tests live in `tests/`; fixtures and reference assets come from `AQUA_tests/` plus the repository-level `cell_area_*.nc` / `weights_*.nc`
- Never create new test data files â€” parametrize over existing grids (`test-r2b0`, `test-pi-2d`, `test-healpix`, `test-hpz3-nested`)
- Use `Reader(model=..., exp=..., source=...)` to load test data; avoid raw `xr.open_dataset` unless testing I/O directly
- Prefer shared fixtures in `tests/conftest.py` (with the appropriate scope) to avoid repeated heavy retrievals
- For expensive retrievals, prefer `session` or `module` scoped `Reader` fixtures; the suite is already structured to reuse reads and reduce xdist/temp-file contention
- Instead of duplicating test bodies (if the test requires it), favor `pytest.mark.parametrize`

## Integration Checklist

- [ ] `log_configure()` used, `loglevel` accepted in constructor
- [ ] All path/config discovery via `ConfigPath` / `ConfigLocator`; no ad hoc search logic or hardcoded strings
- [ ] `to_list()` used for any variable-length input
- [ ] `DataModel.apply()` used for coordinate transformations
- [ ] `xr.set_options(keep_attrs=True)` set; no `copy(deep=True)` unless necessary
- [ ] Operations remain lazy until explicitly needed
- [ ] Semantic data transforms append `log_history()` and preserve `AQUA_*` attrs
- [ ] xarray outputs that should keep AQUA chaining call `data.aqua.set_default(self)`
- [ ] Shared YAML/config writes use `SafeFileLock` + `dump_yaml()`
- [ ] No new dependency added unless clearly asked for by the user
- [ ] Tests reuse `AQUA_tests/` and existing shared grid assets

---

## Current Codebase Divergences (Documented)

- CLI currently includes some user-facing `print()` calls in `aqua/core/console/`; prefer logger in new code, and update existing calls only in files you are already modifying for functional changes
- `xr.set_options(keep_attrs=True)` is currently set in multiple modules (not globally once); preserve this behavior unless a repo-wide consolidation is planned
- Some install/config flows still use explicit defaults (e.g. `config-aqua.yaml`, `$HOME/.aqua`); route new path logic through `ConfigPath` and `ConfigLocator`
